# PostQuantum.DataProtection.Fuzz

SharpFuzz harness for the library's two untrusted-input decoders:

- `HybridKemEnvelope.Decode` — the envelope bytes ASP.NET Core Data Protection pulls off disk.
- `PostQuantumKeyPair.Decode` — the keystore token a corrupted file would hand us.

Both decoders cap length-prefixed fields, use overflow-safe bounds arithmetic, and have
`TryDecode` overloads. The fuzz harness verifies the **contract**: malformed input throws only
`ArgumentException` or `FormatException`, never an unexpected exception type, never an
allocation explosion, never an unbounded loop.

## Running locally (Linux)

```bash
# 1. Instrument the library assembly.
dotnet tool install --global SharpFuzz.CommandLine
dotnet build -c Release
sharpfuzz src/PostQuantum.DataProtection/bin/Release/net10.0/PostQuantum.DataProtection.dll

# 2. Build the fuzz harness.
dotnet build fuzz/PostQuantum.DataProtection.Fuzz -c Release

# 3. Seed corpus from this repo's pinned-format tests (or any real envelope).
mkdir -p corpus
dotnet run -c Release --project samples/AspNetCore.Sample &
sleep 5
curl -s http://localhost:5117/protect-demo > /dev/null
pkill -f AspNetCore.Sample
# now grep an envelope token out of keys/data-protection/key-*.xml into corpus/

# 4. Run afl-fuzz.
afl-fuzz -i corpus -o findings -- \
  dotnet fuzz/PostQuantum.DataProtection.Fuzz/bin/Release/net10.0/PostQuantum.DataProtection.Fuzz.dll envelope
```

Replace `envelope` with `keypair` to fuzz the keypair decoder instead.

## What "interesting" looks like

A finding in `findings/crashes/` means the harness caught an exception type it did not expect
(anything other than `ArgumentException` / `FormatException`). That is a real bug — please open a
security advisory rather than a public issue.

## In CI

We do not run AFL in CI — it is a long-running explorer, not a unit test. The CI suite carries a
**property-based test** (`HostileInputContractTests`) that drives 10 000 randomly-generated byte
arrays through both decoders on every push, asserting the same exception-type contract. That
catches the obvious regressions; AFL catches the obscure ones.
